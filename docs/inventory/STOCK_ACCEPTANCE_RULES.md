## Purpose
Ensure stock acceptance is **verifiable, accountable, and auditable**.

Stock acceptance is a **control gate**, not a formality.

---

## ACCEPTANCE PRE-CONDITIONS

Before acceptance, Store Manager must verify:
1. Physical quantity
2. Barcode presence
3. SKU correctness
4. MRP vs Offer Price logic
5. Location assignment

No partial acceptance without justification.

---

## BARCODE GOVERNANCE

- Barcodes are printed ONLY at store level
- Incoming stock must have:
  - No retail price tags
  - No barcodes from source
- Store prints fresh barcode with:
  - SKU
  - Store code
  - Display location

---

## LOCATION MAPPING

Each accepted item must be assigned:
- Counter / Shelf / Cabinet code
- Example: C1, S3, CAB-2

Location change later:
- Allowed
- New barcode must be printed
- Old barcode auto-invalidated

---

## MISMATCH HANDLING

If mismatch detected:
- Acceptance paused
- Escalation task created
- HQ notified automatically

Store cannot:
- Edit SKU
- Edit MRP
- Edit offer price

---

## FINAL ACCEPTANCE

Once accepted:
- Stock becomes sellable
- Audit clock starts
- Responsibility shifts to store

Acceptance is irreversible without Admin approval.
