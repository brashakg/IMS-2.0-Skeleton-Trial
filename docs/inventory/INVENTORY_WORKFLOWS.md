# INVENTORY WORKFLOWS — IMS 2.0

---

## FLOW 1 — HQ PRODUCT CATALOG → STORE

1. Product created at HQ
2. Attributes defined
3. MRP / Offer locked
4. Product activated
5. Shipped to store
6. Store receives
7. Store Manager verifies
8. Accept OR mismatch
9. Barcode printed
10. Location assigned

---

## FLOW 2 — STORE MISMATCH ESCALATION

1. Store flags issue
2. Uploads images
3. Product quarantined
4. HQ reviews
5. Decision:
   - Correct SKU
   - Replace
   - Return to vendor
6. Resolution logged

---

## FLOW 3 — STOCK TRANSFER STORE → STORE

1. Transfer request created
2. Approval (if required)
3. Source removes barcodes
4. Ships stock
5. Destination receives
6. New barcodes printed
7. Location assigned

---

## FLOW 4 — DAILY STOCK COUNT

1. Staff scans assigned stock
2. System compares expected vs actual
3. Variance flagged
4. Task auto-created
5. Escalation if repeated

---

## FLOW 5 — EMPLOYEE EXIT STOCK CHECK

1. Exit initiated
2. Assigned stock audit forced
3. Clearance mandatory
4. Exit blocked until resolved

---

END OF INVENTORY WORKFLOWS
