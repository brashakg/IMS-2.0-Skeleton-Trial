# CLINICAL WORKFLOWS — IMS 2.0

---

## FLOW 1 — IN-STORE EYE TEST

1. Customer arrives
2. Patient selected or created
3. Optometrist assigned
4. Eye test performed
5. Prescription entered
6. Validity set
7. Prescription saved
8. Version locked

---

## FLOW 2 — EXTERNAL DOCTOR PRESCRIPTION

1. Sales staff selects “From Doctor”
2. Prescription values entered
3. Marked as “External”
4. No optometrist attribution
5. Saved with source tag

---

## FLOW 3 — PRESCRIPTION UPDATE

1. New test required
2. New prescription version created
3. Old version retained
4. Orders reference latest valid version

---

## FLOW 4 — PRESCRIPTION → ORDER

1. Valid prescription selected
2. System checks expiry
3. If expired → block or escalate
4. Order proceeds only if valid

---

## FLOW 5 — COMPLAINT / REDO

1. Complaint logged
2. Prescription linked
3. Redo flag applied
4. Clinical review required
5. Insight logged
