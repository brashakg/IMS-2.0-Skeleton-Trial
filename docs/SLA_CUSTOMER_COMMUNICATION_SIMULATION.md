# SLA & CUSTOMER COMMUNICATION SIMULATION — IMS 2.0

## Purpose
Ensure customers are never forgotten, delayed, or misinformed.

---

## PATTERN 1 — SILENT ORDER DELAYS

Pattern:
- Order status not updated beyond SLA window

Detection:
- Status age vs SLA threshold

Action:
- Auto task for staff
- Escalation to Manager if untouched

---

## PATTERN 2 — MISLEADING PROMISES

Pattern:
- Delivery dates edited repeatedly

Detection:
- Date mutation frequency

Action:
- Admin visibility
- Customer communication audit

---

## PATTERN 3 — CALLBACK NEGLIGENCE

Pattern:
- Repeated missed callbacks

Detection:
- Task closure vs call log mismatch

Action:
- Store Manager task

---

## GOVERNANCE
- ❌ No auto customer messages
- ✅ Human-reviewed communication
- ✅ Full SLA audit trail
